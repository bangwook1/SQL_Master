# 1️⃣ 개념 정리

## 01. 여러 가지 과제 

### 느낀 점

데이터 분석 직무라고 해서 **데이터 분석 자체에만 매몰되는 것이 아니라**,  
현업에서는 협업할 일이 훨씬 많기 때문에  

- 타 직무에 대한 이해  
- 엔지니어와의 협업을 위한 기술적 이해  
- 빅데이터 환경에 대한 전반적인 이해  

와 같은 부분이 중요하겠다는 생각이 들었다.

## 02. 시스템

## PostgreSQL
- 오픈소스 **RDB(Relational Database)**  
- 표준 SQL을 잘 준수  
- Window Function, CTE(WITH) 등 분석용 SQL 기능 지원  
- 다양한 확장 기능 제공  
- 소규모 데이터 분석 및 SQL 학습에 적합

## Apache Hive
- **HDFS 기반 분산 파일 시스템 위에서 동작하는 SQL 처리 시스템**
- HiveQL을 MapReduce 작업으로 변환하여 병렬 처리 수행
- 대용량 데이터 분석에 적합
- 파일 기반 구조라 **레코드 수정/삭제가 어려움**
- 인덱스가 기본적으로 없고 전체 파일 스캔 방식 사용

## Amazon Redshift
- AWS에서 제공하는 **분산 병렬 RDB**
- 대용량 데이터 분석에 최적화
- 빠른 쿼리 응답 속도
- **컬럼 지향 스토리지** 사용
- 클라우드 서비스로 확장성 높지만 사용 비용 발생

## Google BigQuery
- 구글이 제공하는 **클라우드 기반 빅데이터 분석 서비스**
- 서버 관리 없이 SQL로 데이터 분석 가능
- 읽은 데이터 양 기준으로 비용 발생
- Standard SQL 지원
- 컬럼 지향 저장 방식 사용

## SparkSQL
- **Apache Spark의 SQL 인터페이스**
- 분산 데이터 처리 가능
- SQL뿐 아니라 Python, Scala, Java, R 등 다양한 언어 지원
- DataFrame API 제공
- 빅데이터 분석, 머신러닝, 스트리밍 처리까지 가능

## 데이터의 종류



- **업무 데이터**: 서비스와 시스템 운영을 위해 저장되는 데이터
- **로그 데이터**: 분석을 목적으로 수집, 기록되는 데이터

---

## 1. 업무 데이터

서비스 운영을 위해 저장된 데이터  
**갱신형 데이터**라는 특징

### 특징
- 데이터의 정밀도와 정합성이 높음
- 데이터가 추가, 수정, 삭제될 수 있음
- 여러 테이블로 나뉘어 있어 결합이 필요

### 종류
- **트랜잭션 데이터**: 구매, 리뷰, 게임 플레이 등 사용자 행동 기록
- **마스터 데이터**: 회원 정보, 상품 정보, 카테고리 정보 등 기준 정보

### 활용
- 매출액, 사용자 수 등 **정확한 값이 필요한 분석**에 적합
- 다만 데이터가 계속 갱신되므로 **추출 시점에 따라 결과가 달라질 수 있음**

---

## 2. 로그 데이터

분석이나 통계를 목적으로 수집 
**누적형 데이터**라는 특징

### 특징
- 시간, IP, URL, 레퍼러, Cookie 등 다양한 정보 저장 가능
- 과거 데이터가 수정되지 않고 계속 누적됨
- 수집 방식에 따라 데이터 정확도가 달라질 수 있음

### 수집 방식
- **비컨 방식**: 태그나 SDK를 통해 사용자 장치에서 전송
- **서버 방식**: 서버에서 직접 로그 출력

### 활용
- 방문 수, 페이지뷰, 유입 경로, 사용자 행동 분석에 적합
- but 누락이나 크롤러 포함 등으로 인해 **정확도는 업무 데이터보다 낮을 수 있음**

---

## 3. 업무 데이터와 로그 데이터의 차이

| 구분 | 업무 데이터 | 로그 데이터 |
|---|---|---|
| 목적 | 서비스 운영 | 분석 및 통계 |
| 데이터 형태 | 갱신형 | 누적형 |
| 정확도 | 높음 | 상대적으로 낮음 |
| 활용 예시 | 매출, 구매, 회원 수 | 방문 수, 페이지뷰, 유입 분석 |

---

## 4. 두 데이터를 함께 사용할 때의 가치

업무 데이터와 로그 데이터를 함께 사용하면 더 넓은 분석이 가능

### 예시
- 웹사이트 행동이 실제 구매나 계약에 어떤 영향을 주는지 분석 가능
- 온라인 행동과 오프라인 행동의 연결 가능
- 사용자 경향 파악 및 서비스 개선 가능

---

## 5. 데이터 활용의 가치

데이터는 단순히 수집하는 것이 아니라, **서비스와 조직에 가치를 줄 때 의미가 있음**

### 활용 목적
- **목표 관리**: 매출, 사용자 수, 방문 수 등 현황 파악
- **서비스 개선**: 사용자 행동을 분석해 개선점 도출
- **미래 예측**: 이탈 예측, 추천 시스템 등 예측 기반 활용
